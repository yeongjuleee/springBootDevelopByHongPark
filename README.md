### part2course4 : 3장, DB 데이터 조회하기 

#### 3.4.1 H2 DB 접속하기 (128p)
* [H2 DB 접속하기](localhost:8080/h2-console)
  * 이때 JDBC URL에 적힌 값이 DB 접근 주소인데, 서버를 실행할 때 마다 바뀌므로 이 값을 찾아 다시 입력해줘야 접속할 수 있다. 
  * `Run` 탭에서 `Ctrl` + `F` 키를 누르고 jdbc 를 검색한 다음 `url=jdbc:h2:mem:...` 내용을 복사해서 JDBC URL에 넣어주면 접속이 가능하다.
  
* `SELECT` 문은 테이블의 속성을 조회하는 명령 : `SELECT 속성명 FROM 테이블명;` 형식으로 작성한다.  (속성명 대신 `*`를 사용하면 모든 속성을 조회하라는 뜻이다.)
  * 데이터는 테이블에 행 단위로 저장되는데, 이러한 행을 `레코드(record)`라고 한다.
* SQL 문으로 직접 레코드 삽입하기 : `INSERT INTO 테이블명(속성명1, 속성명2, 속성명3, ...) VALUES(값1, 값2, 값3, ...)`

### 3장 마무리 개념 정리
1. 폼 데이터와 `<form>` 태그 : 폼 데이터는 `<form>` 태그에 실려 전송되는 데이터 
   * `<form>` 태그에는 전송할 데이터를 "어디에 보내는가 => `action`" 와 "어떻게 보내는가 => `method`" 속성으로 설정한다.
2. `DTO(Data Transfer Object)` : 서버의 컨트롤러가 데이터를 담는 객체로 최종적으로 데이터베이스(DB)에 저장된다.
3. 데이터베이스(`Database`) : 데이터를 관리하는 창고로, 모든 데이터는 행과 열로 구성된 테이블에 저장되고 관리된다.
4. JPA(`Java Persistence API`) : 자바 언어로 DB에 명령을 내리는 도구
   * 엔티티(`Entity`) : 자바 객체를 DB가 이해할 수 있게 만든 것으로 이를 기반으로 테이블이 만들어진다.
   * 리포지토리(`Repository`) : 엔티티가 DB 속 테이블에 저장 및 관리될 수 있게 하는 인터페이스
5. 의존성 주입(DI, Dependency Injection) : 외부에서 만들어진 객체를 필요한 곳으로 가져오는 기법으로 스프링 부트에서는 `@Autowired`를 이용하여 의존성을 주입한다.
6. SQL(`Structured Query Language`) : 데이터를 관리하기 위해 고안된 DB 언어로 데이터를 생성(Create), 조회(Read), 수정(Update), 삭제(Delete)하는 CRUD 조작을 수행한다.
  * SELECT 문 : `SELECT 속성명 FROM 테이블명;`
  * INSERT 문 : `INSERT INTO 테이블명(속성명1, 속성명2, 속성명3, ...) VALUES (값1, 값2, 값3, ...);`
